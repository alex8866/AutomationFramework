Automation Framework Help

The help contains simplified instructions and links to actions to configure you automation installation.

The web interface to AF is designed to be an AJAX based single page application which is embedded in the automation server. It should run on most modern browsers.

Dashboard
The dashboard contains quick links to most frequent tasks.

The information displayed here contains the IP address of the client, the number of its logs previously stored, its current status and progress, and links to detailed statistics and configuration edit. The statistics and configuration edit button, when clicked, launch a popup.

Server Events
Displays the server logs and the refresh button provides a means to refresh the log content without refreshing the page. The server logs are enabled by redirecting the output to serverlogs.txt as specified in the Makefile.
Configuration backup and restoration
Backing up and restoring a configuration is a good way to ensure that configuration changes dont break due to human errors. Also provides means to quickly replicate a setup.
Global settings
The global settings contain connection parameters used when remote installer is launched. Additionally, it contains profiles available. Profiles are a useful way of separating test cases for different platforms on the same setup. For example a profile for SuSE Linux, and another for ubuntu all derived from the same base root profile.
Remote installer
The remote installer displays the status of remotely managed clients. Additionally, there is a facility to configure the cron jobs for scheduling jobs in the future. These jobs are run daily at the specified hour and allows only 1 schedule per IP to prevent clashes.
